Installation Guide

Deploy OpenCitiVibes for your city or organization in under 5 minutes.

Requirements

Linux server (Ubuntu 22.04+ recommended), Docker & Docker Compose v2, Domain name with DNS configured, 2GB+ RAM (4GB+ for production with PostgreSQL), 20GB+ storage.

Prerequisites

1. Install Docker

If Docker isn't installed on your server:

# Install Docker
curl -fsSL https://get.docker.com | sh

# Add your user to docker group
sudo usermod -aG docker $USER

# Verify installation
docker --version
docker compose version

2. Configure DNS

Point your domain to your server's IP address:

  • yourdomain.com → Your server IP (A record)
  • www.yourdomain.com → Your server IP (A record or CNAME)
  • ntfy.yourdomain.com → Your server IP (for admin notifications)
  • ssh.yourdomain.com → Your server IP (optional, for SSH access)
Cloudflare Users

If using Cloudflare, configure proxy status carefully:

  • Main domain → Proxied (orange cloud) - DDoS protection, CDN
  • ntfy subdomain → DNS only (grey cloud) - WebSocket/SSE needs direct connection
  • ssh subdomain → DNS only (grey cloud) - SSH (port 22) isn't proxied by Cloudflare

Quick Start

Step 1: Clone the Repository

git clone https://github.com/opencitivibes/opencitivibes.git
cd opencitivibes

Step 2: Run Setup Script

# Create a new instance
./scripts/setup-instance.sh montreal idees-montreal.ca

# This creates /opt/opencitivibes-montreal with all necessary files

Step 3: Configure Environment

cd /opt/opencitivibes-montreal

# Edit environment variables
nano .env

Required variables:

# Security - MUST change these!
SECRET_KEY=your-secure-random-key-here
ADMIN_EMAIL=admin@yourdomain.com
ADMIN_PASSWORD=your-secure-admin-password

# Domain
DOMAIN=yourdomain.com

# Database (SQLite for quick start, PostgreSQL for production)
DATABASE_URL=sqlite:///./data/opencitivibes.db
Security Warning

Generate a secure SECRET_KEY: openssl rand -hex 32

Step 4: Configure Platform Identity

nano backend/config/platform.config.json

Minimal configuration:

{
  "platform": {
    "name": "OpenCitiVibes"
  },
  "instance": {
    "name": {
      "en": "Ideas for Montreal",
      "fr": "Idees pour Montreal"
    },
    "entity": {
      "type": "city",
      "name": {
        "en": "Montreal",
        "fr": "Montreal"
      }
    }
  },
  "contact": {
    "email": "contact@yourdomain.com"
  },
  "localization": {
    "default_locale": "en",
    "supported_locales": ["en", "fr"]
  }
}

Step 5: Deploy

Choose the appropriate profile for your environment:

# Development (Mailpit email capture + SQLite)
docker compose --profile dev up -d

# Staging (Postfix SMTP + SQLite)
docker compose --profile staging up -d

# Production (Postfix SMTP + PostgreSQL)
docker compose --profile prod up -d

# Initialize the database
docker compose exec backend python init_db.py

# Check status
docker compose ps

Email Configuration

OpenCitiVibes supports passwordless authentication via email codes. Email configuration depends on your deployment profile:

Development: Mailpit

The dev profile includes Mailpit, which captures all outgoing emails for testing without actually sending them.

# Access inbox at https://yourdomain.com/mailpit/
EMAIL_PROVIDER=smtp
SMTP_HOST=mailpit
SMTP_PORT=1025
SMTP_USE_TLS=false
SMTP_USE_SSL=false
SMTP_FROM_EMAIL=noreply@yourdomain.com
SMTP_FROM_NAME=OpenCitiVibes

Staging/Production: Postfix SMTP

The staging and prod profiles include Postfix, a self-hosted SMTP relay with automatic DKIM signing.

EMAIL_PROVIDER=smtp
SMTP_HOST=postfix
SMTP_PORT=25
SMTP_USE_TLS=false
SMTP_USE_SSL=false
SMTP_FROM_EMAIL=admin@yourdomain.com
SMTP_FROM_NAME=Your Instance Name
MAIL_DOMAIN=yourdomain.com

DNS Records for Email Deliverability

Configure these DNS records to ensure emails aren't marked as spam:

1. SPF Record (TXT on root domain)

v=spf1 ip4:YOUR_SERVER_IP -all

2. DKIM Record

# Get DKIM public key from container
docker exec ${CONTAINER_PREFIX}-postfix cat /etc/opendkim/keys/yourdomain.com/mail.txt

# Add output as TXT record for mail._domainkey.yourdomain.com

3. DMARC Record (TXT on _dmarc)

v=DMARC1; p=quarantine; rua=mailto:admin@yourdomain.com

External SMTP Services

You can also use external SMTP services like SendGrid, Mailgun, etc.:

EMAIL_PROVIDER=smtp
SMTP_HOST=smtp.sendgrid.net
SMTP_PORT=587
SMTP_USE_TLS=true
SMTP_USE_SSL=false
SMTP_USER=apikey
SMTP_PASSWORD=your-sendgrid-api-key
SMTP_FROM_EMAIL=noreply@yourdomain.com
SMTP_FROM_NAME=Your Instance Name
Firewall Note

Many cloud providers block outbound port 25 by default. Check with your VPS provider if emails aren't being delivered. Ensure sudo ufw allow out 25/tcp is set.

Adding Languages

  1. Create translation file: frontend/src/i18n/locales/es.json
  2. Copy structure from en.json
  3. Translate all keys
  4. Add to supported_locales in config

  }

SSL Certificates

Using Let's Encrypt (Recommended)

# Install certbot
sudo apt install certbot

# Get certificate for main domain AND ntfy subdomain
sudo certbot certonly --standalone \
  -d yourdomain.com \
  -d ntfy.yourdomain.com

# Copy certificates to deploy directory
sudo cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem ssl/
sudo cp /etc/letsencrypt/live/yourdomain.com/privkey.pem ssl/
sudo chown $USER:$USER ssl/*

# Auto-renewal is configured automatically via certbot.timer

Using the Setup Script

# Initial setup (requests certs for domain and ntfy.domain)
./scripts/setup-ssl.sh yourdomain.com

# Renewal (run periodically or via cron)
./scripts/setup-ssl.sh --renew yourdomain.com

Backups

# Manual backup
./scripts/backup.sh

# Backups stored in /opt/opencitivibes-instance/backups/

# Restore from backup
./scripts/restore.sh backups/backup-2024-01-15.tar.gz

Updates

# Pull latest images (use your profile)
docker compose --profile prod pull

# Restart with new images
docker compose --profile prod up -d

# Run migrations if needed
docker compose exec backend alembic upgrade head

Push Notifications

OpenCitiVibes includes a self-hosted ntfy server for admin push notifications.

Enable Notifications

# Add to .env
NTFY_URL=http://ntfy:80
NTFY_TOPIC_PREFIX=admin
NTFY_ENABLED=true
APP_URL=https://yourdomain.com
NTFY_BASE_URL=https://ntfy.yourdomain.com

# Optional: Set a custom password for ntfy admin user
# NTFY_ADMIN_PASSWORD=YourSecurePassword

Set Up Admin Access

The seed-admin.sh script automatically creates the ntfy admin user. If you need to set it up manually:

# Create admin user with password (avoid special characters like !)
docker exec -e NTFY_PASSWORD="YourSecurePass123" \
  ${CONTAINER_PREFIX}-ntfy ntfy user add --role=admin admin

# Configure anonymous write access for backend to publish notifications
docker exec ${CONTAINER_PREFIX}-ntfy ntfy access '*' "admin-*" write

# Create auth token for backend to read notification history
docker exec ${CONTAINER_PREFIX}-ntfy ntfy token add --label=backend admin
# Add the token output to .env as NTFY_AUTH_TOKEN=tk_xxxxx
Password Note

Avoid special characters like ! in the ntfy password as they can cause authentication issues with some clients.

Mobile Apps

Receive instant push notifications on your mobile device using the official ntfy apps:

Download ntfy

iOS: App Store
Android: Google Play or F-Droid

App Setup

  1. Open the ntfy app and tap Settings (gear icon)
  2. Tap Add default server or Manage servers
  3. Enter your self-hosted server URL: https://ntfy.yourdomain.com
  4. If authentication is enabled, enter your admin username and password
  5. Return to the main screen and tap + to subscribe to a topic

Recommended Topics

Topic Notifications
admin-ideas New idea submissions requiring moderation
admin-appeals User appeals for rejected content
admin-reports Flagged content reports

App Features

  • Instant delivery: Push notifications arrive immediately via Firebase (Android) or APNs (iOS)
  • Background sync: Receive notifications even when the app is closed
  • Action buttons: Tap notification actions to approve/reject directly (if configured)
  • Offline support: Missed notifications are delivered when you reconnect
  • Multiple servers: Connect to multiple ntfy instances from one app

Troubleshooting

Services not starting

# Check container status
docker compose ps

# View detailed logs
docker compose logs backend
docker compose logs frontend

Database connection errors

# Verify DATABASE_URL in .env
# Ensure data directory exists
ls -la data/

# Check permissions
chmod 755 data/

SSL certificate issues

# Check nginx config
docker compose exec nginx nginx -t

# Renew certificate
sudo certbot renew

Push notifications not working

# Check ntfy container
docker compose ps ntfy

# Test connectivity
docker exec ${CONTAINER_PREFIX}-backend curl -s http://ntfy:80/v1/health

# Check environment variables
docker exec ${CONTAINER_PREFIX}-backend env | grep NTFY

Email/login codes not working

# Check email configuration
docker exec ${CONTAINER_PREFIX}-backend env | grep -E "EMAIL|SMTP"

# Check Postfix container (staging/prod)
docker compose ps postfix
docker logs ${CONTAINER_PREFIX}-postfix --tail 50

# Test SMTP connectivity
docker exec ${CONTAINER_PREFIX}-backend python -c "
import smtplib
s = smtplib.SMTP('postfix', 25)
print(s.noop())
s.quit()
"

# Check if port 25 is blocked
nc -zv smtp.gmail.com 25 || echo "Port 25 blocked by provider"
