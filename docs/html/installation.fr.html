Guide d'installation

Déployez OpenCitiVibes pour votre ville ou organisation en moins de 5 minutes.

Prérequis

Serveur Linux (Ubuntu 22.04+ recommandé), Docker et Docker Compose v2, nom de domaine avec DNS configuré, 2 Go+ RAM (4 Go+ pour production avec PostgreSQL), 20 Go+ stockage.

Prérequis

1. Installer Docker

Si Docker n'est pas encore installé sur votre serveur :

# Installer Docker
curl -fsSL https://get.docker.com | sh

# Ajouter votre utilisateur au groupe docker
sudo usermod -aG docker $USER

# Vérifier l'installation
docker --version
docker compose version

2. Configurer le DNS

Pointez votre domaine vers l'adresse IP de votre serveur :

  • votredomaine.com → IP de votre serveur (enregistrement A)
  • www.votredomaine.com → IP de votre serveur (A ou CNAME)
  • ntfy.votredomaine.com → IP de votre serveur (pour les notifications admin)
Configuration proxy Cloudflare

Si vous utilisez Cloudflare, configurez les paramètres proxy (nuage orange/gris) appropriés :

  • votredomaine.com - Proxié (nuage orange) pour la protection DDoS et le cache
  • www.votredomaine.com - Proxié (nuage orange)
  • ntfy.votredomaine.com - DNS uniquement (nuage gris) - requis pour WebSocket/SSE
  • ssh.votredomaine.com - DNS uniquement (nuage gris) - si vous voulez accéder via SSH sur un sous-domaine

Le sous-domaine ntfy doit être DNS uniquement car le proxy Cloudflare interfère avec les connexions WebSocket et Server-Sent Events utilisées par ntfy.

Démarrage rapide

Étape 1 : Cloner le dépôt

git clone https://github.com/opencitivibes/opencitivibes.git
cd opencitivibes

Étape 2 : Exécuter le script de configuration

# Créer une nouvelle instance
./scripts/setup-instance.sh montreal idees-montreal.ca

# Cette commande crée /opt/opencitivibes-montreal avec tous les fichiers nécessaires

Étape 3 : Configurer l'environnement

cd /opt/opencitivibes-montreal

# Modifier les variables d'environnement
nano .env

Variables requises :

# Sécurité - VOUS DEVEZ changer ces valeurs!
SECRET_KEY=votre-cle-aleatoire-securisee-ici
ADMIN_EMAIL=admin@votredomaine.com
ADMIN_PASSWORD=votre-mot-de-passe-admin-securise

# Domaine
DOMAIN=votredomaine.com

# Base de données (SQLite pour commencer, PostgreSQL pour la production)
DATABASE_URL=sqlite:///./data/opencitivibes.db
Important

Générez une SECRET_KEY sécurisée : openssl rand -hex 32

Étape 4 : Configurer les informations de la plateforme

nano backend/config/platform.config.json

Configuration minimale :

{
  "platform": {
    "name": "OpenCitiVibes"
  },
  "instance": {
    "name": {
      "en": "Ideas for Montreal",
      "fr": "Idées pour Montréal"
    },
    "entity": {
      "type": "city",
      "name": {
        "en": "Montreal",
        "fr": "Montréal"
      }
    }
  },
  "contact": {
    "email": "contact@votredomaine.com"
  },
  "localization": {
    "default_locale": "fr",
    "supported_locales": ["fr", "en"]
  }
}

Étape 5 : Déployer

Choisissez le profil approprié pour votre environnement :

# Développement (capture courriel Mailpit + SQLite)
docker compose --profile dev up -d

# Pré-production (SMTP Postfix + SQLite)
docker compose --profile staging up -d

# Production (SMTP Postfix + PostgreSQL)
docker compose --profile prod up -d

# Initialiser la base de données
docker compose exec backend python init_db.py

# Vérifier le statut
docker compose ps

Configuration courriel

OpenCitiVibes supporte l'authentification sans mot de passe via codes courriel. La configuration dépend de votre profil de déploiement :

Développement : Mailpit

Le profil dev inclut Mailpit, qui capture tous les courriels sortants pour les tests sans les envoyer réellement.

# Accéder à la boîte de réception à https://votredomaine.com/mailpit/
EMAIL_PROVIDER=smtp
SMTP_HOST=mailpit
SMTP_PORT=1025
SMTP_USE_TLS=false
SMTP_USE_SSL=false
SMTP_FROM_EMAIL=noreply@votredomaine.com
SMTP_FROM_NAME=OpenCitiVibes

Pré-production/Production : SMTP Postfix

Les profils staging et prod incluent Postfix, un relais SMTP auto-hébergé avec signature DKIM automatique.

EMAIL_PROVIDER=smtp
SMTP_HOST=postfix
SMTP_PORT=25
SMTP_USE_TLS=false
SMTP_USE_SSL=false
SMTP_FROM_EMAIL=admin@votredomaine.com
SMTP_FROM_NAME=Nom de votre instance
MAIL_DOMAIN=votredomaine.com

Enregistrements DNS pour la délivrabilité

Configurez ces enregistrements DNS pour éviter que les courriels soient marqués comme spam :

1. Enregistrement SPF (TXT sur le domaine racine)

v=spf1 ip4:VOTRE_IP_SERVEUR -all

2. Enregistrement DKIM

# Obtenir la clé publique DKIM du conteneur
docker exec ${CONTAINER_PREFIX}-postfix cat /etc/opendkim/keys/votredomaine.com/mail.txt

# Ajouter la sortie comme enregistrement TXT pour mail._domainkey.votredomaine.com

3. Enregistrement DMARC (TXT sur _dmarc)

v=DMARC1; p=quarantine; rua=mailto:admin@votredomaine.com

Services SMTP externes

Vous pouvez aussi utiliser des services SMTP externes comme SendGrid, Mailgun, etc. :

EMAIL_PROVIDER=smtp
SMTP_HOST=smtp.sendgrid.net
SMTP_PORT=587
SMTP_USE_TLS=true
SMTP_USE_SSL=false
SMTP_USER=apikey
SMTP_PASSWORD=votre-cle-api-sendgrid
SMTP_FROM_EMAIL=noreply@votredomaine.com
SMTP_FROM_NAME=Nom de votre instance
Note pare-feu

Plusieurs fournisseurs cloud bloquent le port sortant 25 par défaut. Vérifiez auprès de votre fournisseur VPS si les courriels ne sont pas livrés. Assurez-vous que sudo ufw allow out 25/tcp est configuré.

Certificats SSL

Utiliser Let's Encrypt (recommandé)

OpenCitiVibes inclut un script d'aide pour configurer les certificats SSL pour le domaine principal et le sous-domaine ntfy :

# Configuration initiale (arrête nginx, obtient le certificat)
./scripts/setup-ssl.sh votredomaine.com

# Cela obtient des certificats pour :
#   - votredomaine.com
#   - ntfy.votredomaine.com

# Renouvellement (quand les certificats expirent)
./scripts/setup-ssl.sh --renew votredomaine.com

# Utiliser le mode webroot si nginx est en cours d'exécution
./scripts/setup-ssl.sh --webroot votredomaine.com

Configuration manuelle

# Installer certbot
sudo apt install certbot

# Obtenir le certificat (inclut le sous-domaine ntfy)
sudo certbot certonly --standalone \
    -d votredomaine.com \
    -d ntfy.votredomaine.com

# Copier les certificats vers le répertoire de déploiement
sudo cp /etc/letsencrypt/live/votredomaine.com/fullchain.pem ssl/
sudo cp /etc/letsencrypt/live/votredomaine.com/privkey.pem ssl/

# Renouvellement automatique via cron
0 0 1 * * cd /opt/opencitivibes-instance && ./scripts/setup-ssl.sh --renew votredomaine.com

# Sauvegarde manuelle

Notifications push

OpenCitiVibes inclut un serveur ntfy hébergé localement pour les notifications push admin.

Activer les notifications

# Ajouter au .env
NTFY_URL=http://ntfy:80
NTFY_TOPIC_PREFIX=admin
NTFY_ENABLED=true
APP_URL=https://votredomaine.com
NTFY_BASE_URL=https://ntfy.votredomaine.com

Configurer l'authentification ntfy

Par défaut, ntfy utilise auth-default-access: "deny-all" pour la sécurité. Vous devez créer des utilisateurs et un jeton d'authentification :

# Créer un utilisateur admin pour les abonnements mobile app
NTFY_ADMIN_PASS="VotreMotDePasseSecurise123"
docker exec -e NTFY_PASSWORD="${NTFY_ADMIN_PASS}" ${CONTAINER_PREFIX}-ntfy ntfy user add --role=admin admin

# Créer un jeton d'authentification pour que le backend puisse lire l'historique des notifications
NTFY_TOKEN=$(docker exec ${CONTAINER_PREFIX}-ntfy ntfy token add --label=backend admin 2>/dev/null | grep -oP 'token \K[^ ]+')
echo "NTFY_AUTH_TOKEN=${NTFY_TOKEN}" >> .env

# Redémarrer le backend pour récupérer le nouveau jeton
docker compose --profile prod up -d backend
Important : Caractères de mot de passe

Évitez les caractères spéciaux comme ! dans le mot de passe admin ntfy car ils peuvent causer des problèmes d'authentification. Utilisez des mots de passe alphanumériques avec des chiffres (ex: SecurePass123).

Applications mobiles

Recevez des notifications push instantanées sur votre appareil mobile avec les applications officielles ntfy :

Télécharger ntfy

iOS : App Store
Android : Google Play ou F-Droid

Configuration de l'application

  1. Ouvrez l'app ntfy et appuyez sur Paramètres (icône d'engrenage)
  2. Appuyez sur Ajouter un serveur par défaut ou Gérer les serveurs
  3. Entrez l'URL de votre serveur hébergé localement : https://ntfy.votredomaine.com
  4. Si l'authentification est activée, entrez votre nom d'utilisateur et mot de passe admin
  5. Revenez à l'écran principal et appuyez sur + pour vous abonner à un topic

Topics recommandés

Topic Notifications
admin-ideas Nouvelles soumissions d'idées à modérer
admin-appeals Appels des utilisateurs pour contenu rejeté
admin-reports Signalements de contenu

Fonctionnalités de l'application

  • Livraison instantanée : Les notifications push arrivent immédiatement via Firebase (Android) ou APNs (iOS)
  • Synchronisation en arrière-plan : Recevez les notifications même quand l'app est fermée
  • Boutons d'action : Appuyez sur les actions de notification pour approuver/rejeter directement (si configuré)
  • Support hors ligne : Les notifications manquées sont livrées à la reconnexion
  • Serveurs multiples : Connectez-vous à plusieurs instances ntfy depuis une seule app

Dépannage

Services qui ne démarrent pas

# Vérifier le statut des conteneurs
docker compose ps

# Voir les logs détaillés
docker compose logs backend
docker compose logs frontend

Erreurs de connexion base de données

# Vérifier DATABASE_URL dans .env
# S'assurer que le répertoire data existe
ls -la data/

# Vérifier les permissions
chmod 755 data/

Problèmes de certificat SSL

# Vérifier la config nginx
docker compose exec nginx nginx -t

# Renouveler le certificat
sudo certbot renew

Notifications push qui ne fonctionnent pas

# Vérifier le conteneur ntfy
docker compose ps ntfy

# Tester la connectivité
docker exec ${CONTAINER_PREFIX}-backend curl -s http://ntfy:80/v1/health

# Vérifier les variables d'environnement
docker exec ${CONTAINER_PREFIX}-backend env | grep NTFY

Courriels/codes de connexion qui ne fonctionnent pas

# Vérifier la configuration courriel
docker exec ${CONTAINER_PREFIX}-backend env | grep -E "EMAIL|SMTP"

# Vérifier le conteneur Postfix (staging/prod)
docker compose ps postfix
docker logs ${CONTAINER_PREFIX}-postfix --tail 50

# Tester la connectivité SMTP
docker exec ${CONTAINER_PREFIX}-backend python -c "
import smtplib
s = smtplib.SMTP('postfix', 25)
print(s.noop())
s.quit()
"

# Vérifier si le port 25 est bloqué
nc -zv smtp.gmail.com 25 || echo "Port 25 bloqué par le fournisseur"
